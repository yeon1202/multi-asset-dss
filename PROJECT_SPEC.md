# 다자산 의사결정 지원 시스템 (Multi-Asset Decision Support System)

> **승연 님의 개인 투자 어드바이저 — 자동 주문이 아니라, 자동 분석·추천 시스템**

이 문서는 Claude Code가 프로젝트를 이해하고 단계별로 구축할 수 있도록 작성된 사양서입니다. 항상 이 문서의 **철학과 제약**을 우선시한 뒤 코드를 작성해 주세요.

---

## 0. 가장 먼저 — Claude Code가 지켜야 할 원칙

1. **이 시스템은 분석·추천만 한다. 실제 주문은 절대 자동 실행하지 않는다.**
2. **검증 없는 코드를 생산하지 않는다.** 모든 모듈은 단위 테스트 또는 출력 검증 코드를 함께 작성.
3. **하드코딩 금지.** 모든 종목 리스트, 임계값, 가중치는 `config/` 디렉토리 설정 파일로 분리.
4. **점진적 개발.** Phase 1을 완성하고 동작을 확인한 뒤에야 Phase 2로 넘어간다.
5. **승연 님은 코딩 학습 중인 초보자다.** 새로운 라이브러리·함수·문법을 처음 사용할 때는 주석으로 간단히 설명한다.
6. **숫자는 거짓말을 한다.** 백테스팅 결과는 항상 수수료·슬리피지·세금을 반영. 그렇지 않은 결과는 사용 금지.

## 1. 프로젝트 개요

### 1.1 목표
다양한 입력(가격, 기술적 지표, 펀더멘털, 거시 환경, 정성 정보)을 종합해 **"오늘 어떤 자산에 예산을 어떻게 배분할 것인가"** 에 대한 추천을 생성하는 시스템.

### 1.2 핵심 차별점
- ✅ 주식뿐 아니라 **다자산** (ETF, 채권 ETF, 금, 원자재 ETF, 현금)
- ✅ **추천**까지만. 실행은 사용자가.
- ✅ **이유와 함께** 추천 (블랙박스 금지)
- ✅ **확률은 신중하게** — "67% 확률" 대신 "유사 조건 N=82에서 평균 +2.1%, 승률 58%, 표준편차 4.3%" 형식

### 1.3 비-목표 (Non-Goals)
- ❌ HFT, 초단기 매매
- ❌ 실시간 자동 주문 실행
- ❌ 파생상품 (옵션, 선물, 인버스·레버리지 ETF)
- ❌ 신용·미수 거래

## 2. 핵심 철학

### 2.1 퀀터멘털 + LLM (Quantamental + LLM)
- **정량(Quantitative)**: 가격, 재무 비율, 거시 지표 → 일반 코드로 처리
- **정성(Qualitative)**: 공시 원문, 어닝콜, 뉴스 → LLM(Claude API)으로 처리
- 두 결과를 **가중 합산**해 통합 점수 생성

### 2.2 의사결정 지원, 자동 실행 아님
- 시스템은 "이렇게 배분하는 게 유리합니다"까지만 출력
- 실제 주문은 사용자가 증권사 앱에서 직접 수행
- 이유: (1) 안전성, (2) 학습 효과, (3) 법적/기술적 부담 최소화

### 2.3 코어-위성 구조 지원
- **코어**: 인덱스 ETF 적립식 (시스템은 추적만)
- **위성**: 시스템 추천에 따라 능동 운용
- 시스템은 코어와 위성을 **별도 포트폴리오**로 관리

## 3. 시스템 아키텍처 (6 Layers)

                          ↓

## 6. 단계별 개발 로드맵

### Phase 1 (MVP) — "다자산 모니터" ⭐ 여기부터 시작

**목표**: 한국 ETF + 미국 ETF + 금 가격을 수집해 기술적 지표와 함께 보여주는 대시보드.

**완료 조건**:
- [ ] 정해진 자산 리스트의 1년치 가격 데이터를 받아 CSV로 저장
- [ ] RSI, 20일/60일 이동평균, 변동성 계산
- [ ] 각 자산을 정규화한 가격 차트를 Streamlit 대시보드에 표시
- [ ] 자산별 현재 RSI·변동성을 표로 표시
- [ ] `pytest`로 지표 계산 함수의 단위 테스트 통과

**Phase 1 자산 유니버스 (예시)**:
- KODEX 200 (한국 대형주)
- TIGER 미국S&P500
- TIGER 미국나스닥100
- KOSEF 국고채10년
- KODEX 골드선물(H)
- KODEX 미국달러선물

**예상 소요**: 1~2주

---

### Phase 2 — "정량 펀더멘털 스코어링"

**목표**: 개별 종목에 대해 펀더멘털 점수를 자동 계산.

**완료 조건**:
- [ ] DART API로 KOSPI 200 종목의 재무제표 수집
- [ ] PER, PBR, ROE, 영업이익률, 부채비율 계산
- [ ] 종목별 펀더멘털 점수(0~100) 산출
- [ ] 점수 상위 N개 종목 자동 리스트업

**예상 소요**: 2~3주

---

### Phase 3 — "거시 + 시장 국면"

**목표**: 거시 데이터를 받아 시장 국면을 판단.

**완료 조건**:

- [ ] 간단한 룰 기반 레짐 분류기 (Risk-On / Risk-Off / Neutral)
- [ ] 자산별 레짐 적합도 점수 추가

**예상 소요**: 2주

---

### Phase 4 — "포트폴리오 최적화"

**목표**: 통합 점수를 입력으로 받아 실제 비중을 산출.

**완료 조건**:
- [ ] PyPortfolioOpt로 마코위츠 평균-분산 최적화 구현
- [ ] 켈리 공식(하프 켈리) 적용
- [ ] 제약 조건: 한 자산 최대 비중, 현금 최소 비중
- [ ] 일일 리포트 자동 생성 (Markdown)

**예상 소요**: 2~3주

---

### Phase 5 — "백테스팅"

**목표**: 전 모듈을 결합한 전략의 과거 성과 검증.

**완료 조건**:
- [ ] 2015~현재까지 일별 시뮬레이션
- [ ] 수수료(0.015%) + 슬리피지(0.05%) + 세금 반영
- [ ] 성과 지표: 누적 수익률, 샤프지수, MDD(최대낙폭), 승률
- [ ] 벤치마크(KODEX 200 단순 적립) 대비 비교

**예상 소요**: 3~4주

**목표**: 공시·뉴스를 Claude API로 자동 분석.

**완료 조건**:

- [ ] 정성 점수를 통합 스코어링에 가중 반영
- [ ] 비용 모니터링 대시보드

**예상 소요**: 4~6주

### Phase 7 — "알림 & 자동화"

**목표**: 매일 장 마감 후 자동 실행 + 알림.

**완료 조건**:
- [ ] cron 또는 GitHub Actions로 일일 자동 실행
- [ ] 리포트 텔레그램/이메일 전송
- [ ] 큰 변화 발생 시 즉시 알림

**예상 소요**: 1~2주

## 7. 데이터 정책

### 7.1 캐싱
- 같은 데이터를 하루에 두 번 이상 다운로드하지 않음
- `data/raw/` 디렉토리에 날짜별 CSV로 저장
- 캐시 만료 정책: 가격 데이터 1일, 재무 데이터 90일

### 7.2 시점 정합성 (Point-in-Time)
- 백테스팅 시 **그 시점에 알 수 없었던 데이터를 사용하지 않음** (`look-ahead bias` 금지)
- 예: 2020년 백테스팅에서 2021년 수정된 재무제표 사용 금지

### 7.3 결측치 처리
- 결측치를 0으로 채우지 않음 (반드시 NaN으로 유지 또는 forward-fill 명시)

## 8. 보안 & 개인정보

- API 키는 절대 코드에 하드코딩하지 않음 → `.env` 파일 사용
- `.env`는 반드시 `.gitignore`에 포함
- 어떤 경우에도 **증권 계좌 비밀번호, 인증서, 거래 비밀번호** 를 시스템에 저장하지 않음
- 자동 주문 기능은 이 시스템에 **포함되지 않음**

## 9. 제약사항 & 주의점

### 9.1 통계적 함정 회피
- **과적합(overfitting)** 경고: 백테스팅에서 너무 좋은 결과(연 50%+)가 나오면 의심
- **생존자 편향(survivorship bias)**: 현재 상장 종목만 사용하면 결과 왜곡됨 — 가능하면 상장폐지 종목도 포함
- **다중검정(multiple testing)**: 100개 전략 중 하나가 좋아 보여도 우연일 수 있음

### 9.2 LLM 사용 시 (Phase 6+)
- 모든 LLM 출력은 **원본 자료와 교차 검증**
- 환각 방지를 위해 항상 RAG 사용
- 비용 모니터링 필수 (월 한도 설정)
- LLM 출력만으로 추천 점수를 결정하지 않음 — 정량 점수와 결합

### 9.3 사용자에 대한 책임
- 모든 추천에는 **근거** 표시 필수
- "확률 67%" 같은 단정적 표현 사용 금지 → "유사 조건 표본 N개에서 평균 X, 승률 Y" 형식
- 모든 리포트 첫 줄에 면책 문구: "이 리포트는 정보 제공 목적이며, 투자 결정과 그 결과는 본인 책임입니다."

## 10. Phase 1 첫 작업 지시 (Claude Code에게)

승연 님이 이 파일과 함께 "Phase 1을 시작해줘"라고 요청하면, Claude Code는 다음 순서로 작업합니다:

1. **프로젝트 초기화**
   - 위 디렉토리 구조 생성

## 11. 코드 스타일 가이드

- **Type hints** 사용 권장 (`def calc_rsi(prices: pd.Series, period: int = 14) -> pd.Series:`)
- **Docstring**: 함수 첫 줄에 한 줄 한국어 설명, 그 아래 영어 상세 설명 또는 그대로 한국어
- **변수명**: 영어, snake_case. 단 한국 금융 고유 용어는 그대로 사용 가능 (`oegugin_sungmesu` 보다는 `foreign_net_buy` 권장)
- **함수 길이**: 가능하면 50줄 이내. 길어지면 분리.
- **주석**: 새로운 라이브러리·문법은 한 줄 주석으로 설명 (승연 님 학습용)
- **로깅**: `print()` 대신 `loguru` 사용
